## 6. Cross-Component Issues

### 6.1 Column Name Inconsistency (WordPress + Drupal)
Both the WordPress plugin and Drupal module have analytics table column name mismatches between the schema definition and query code. This is a systemic issue — the analytics feature likely has never worked correctly on fresh installs.

### 6.2 Annotations Security (WordPress + Drupal + React)
All three platforms have open/unauthenticated annotation endpoints. This is a cross-platform design issue in the Pro+ tier.

### 6.3 Test Coverage Gaps
- WordPress Plugin: No vendor/PHPUnit installed, 3 missing test files vs documentation claims
- Drupal Module: No vendor installed, tests are trivial mock-free assertions
- React Packages: Tests validate local mocks, not actual code imports
- License Manager: Zero tests

### 6.4 License Validation
- WordPress Plugin: DB hit on every call, no caching
- Drupal Module: Remote validation removed (local-only), which is correct
- React Packages: Client-side validation trivially bypassable
- License Manager: Missing email/encryption classes break the flow
