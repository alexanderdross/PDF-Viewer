## 1. Critical Issue Found & Fixed: WordPress Plugin Installation Error

### Problem
User reported: *"Plugin file does not exist"* error when installing the new WordPress plugin version.

### Root Cause
**Distribution zip folder name mismatch.** The zip file `pdf-embed-seo-optimize-v1.2.12.zip` extracted to a folder named `wordpress-pdf-embed-seo/`, but WordPress expected the plugin at `pdf-embed-seo-optimize/pdf-embed-seo-optimize.php` (matching the text domain and previously registered basename).

### Fix Applied
- **Rebuilt distribution zip** (`dist/pdf-embed-seo-optimize-v1.2.12.zip`) with correct root folder name `pdf-embed-seo-optimize/`
- Verified zip structure now matches WordPress expectations

### Secondary Issue: Activation Hook
The activation hook in `pdf-embed-seo-optimize.php:248-295` called `deactivate_plugins()` before `wp_die()` during version checks. This corrupts WordPress plugin state tracking.

**Fix:** Removed `deactivate_plugins()` calls from activation hook and added `back_link => true` parameter to `wp_die()` calls for better UX.
